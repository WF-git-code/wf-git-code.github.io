汇编

1. 汇编是什么

汇编语言是CPU 机器码的助记符，和机器指令一一对应。CPU 只认 0 和 1 组成的机器码，汇编只是给人看的易读版本。

2. 三大核心存储区域

存储位置 速度 容量 用途
寄存器 极快 极小（几十个） 暂存运算数据、栈指针、指令指针
栈内存 快 有限（几 MB） 函数参数、局部变量、返回地址
全局堆内存 慢 极大 动态分配的数据、全局变量

3. 指令执行流程

CPU 不断重复三步：

  1. 取指：从指令指针指向的内存读取机器码
  2. 译码：翻译这条指令要做什么
  3. 执行：运算、读写内存、跳转等
  • 指令指针寄存器（X86 叫 EIP，X64 叫 RIP，ARM 叫 PC）永远指向下一条即将执行的指令，程序运行时它自动向前移动。

4. 标志位

CPU 运算后会自动更新标志位寄存器，条件跳转指令靠标志位判断是否跳转，是所有 if/else、循环的底层基础。

X86（32 位）汇编基础

32 位是汇编入门的最佳起点，逻辑规整、栈帧清晰，学会后再看 64 位会非常轻松。

一、寄存器体系

X86 有 8 个通用 32 位寄存器，每个都有约定俗成的用途：

寄存器 全称 核心用途（逆向必记）
EAX 累加寄存器 函数返回值默认存在这里；乘除运算默认使用
ECX 计数寄存器 循环计数器；fastcall 约定下传第 1 个参数
EDX 数据寄存器 乘除扩展；fastcall 约定下传第 2 个参数
EBX 基址寄存器 保存全局变量基址，函数内一般不修改
ESI 源索引寄存器 字符串 / 内存拷贝的源地址
EDI 目标索引寄存器 字符串 / 内存拷贝的目标地址
EBP 栈基址寄存器 指向当前栈帧底部，用来寻址局部变量和参数
ESP 栈顶指针寄存器 永远指向当前栈顶，push/pop 会自动改变它的值

补充：EIP（指令指针）不能直接修改，只能通过 call、jmp、ret 这类跳转指令间接改变。

二、标志寄存器 EFLAGS

逆向只需要重点记住 4 个最常用的：

  • ZF（零标志）：运算结果为 0 时置 1。cmp a,b 相等则 ZF=1，对应 je / jz 跳转
  • CF（进位标志）：无符号数运算溢出 / 借位时置 1
  • SF（符号标志）：运算结果为负时置 1
  • OF（溢出标志）：有符号数运算溢出时置 1

X64（64 位）汇编基础

Windows 逆向当前的主力架构，也是你面试的核心重点。和 32 位差异很大，最核心的变化是寄存器更多、调用约定完全重构。

一、核心变化

  1. 所有 32 位寄存器扩展为 64 位，前缀从 E 变 R：EAX→RAX，ESP→RSP，以此类推
  2. 新增 8 个通用寄存器：R8 ~ R15
  3. 取消了多种调用约定，Windows 全平台统一使用 Microsoft x64 调用约定
  4. 绝大多数编译器默认省略栈帧指针（EBP/RBP），直接用 RSP 寻址

二、寄存器体系

寄存器 核心约定用途（逆向必背）
RAX 函数返回值；乘除运算默认寄存器
RCX 第 1 个整型 / 指针参数
RDX 第 2 个整型 / 指针参数
R8 第 3 个整型 / 指针参数
R9 第 4 个整型 / 指针参数
R10、R11 临时寄存器，函数内可随意修改（易失）
R12~R15、RBX、RDI、RSI 非易失寄存器，函数要用必须先保存、返回前恢复
RSP 栈顶指针，永远指向栈顶
RBP 可选栈基址，Release 版通常不用
RIP 指令指针，指向下一条指令

三、Microsoft x64 调用约定（面试核心考点）

这是 Windows x64 唯一的标准调用约定，所有系统 API、编译器生成代码都遵守它。

1. 参数传递规则

  • 前 4 个整型 / 指针参数：依次放入 RCX、RDX、R8、R9
  • 第 5 个及以后的参数：从右往左压入栈中
  • 浮点参数对应放入 XMM0 ~ XMM3

2. 影子空间（Shadow Space）

这是 x64 最特殊、新手最懵的知识点：

  • 定义：调用函数前，调用者必须在栈上预留 32 字节（4 个 8 字节） 的空间
  • 位置：紧跟在返回地址的下方
  • 用途：给被调用函数临时存放 4 个寄存器参数，方便调试、变参函数使用
  • 规则：哪怕函数只有 1 个参数，也必须预留完整的 32 字节；由调用者分配、调用者回收

3. 栈 16 字节对齐

调用 call 指令前，RSP 必须是 16 字节对齐的，否则调用系统 API 会直接崩溃。

4. 返回值规则

  • 普通整型 / 指针返回值放在 RAX
  • 64 位以上的大返回值，由调用者传指针当第一个参数（RCX）

六、Release 版特点

  • 绝大多数情况省略 RBP 栈帧，所有局部变量、参数都用 RSP+偏移 寻址
  • 大量函数内联，看不到明显的 call 调用
  • 寄存器优化充分，变量尽量放寄存器不存栈

ARM 汇编基础

多用于安卓逆向、嵌入式、IoT 设备逆向，属于拓展内容，掌握核心差异即可。

一、架构核心特点

  1. RISC 精简指令集：指令数量少、格式规整，单条指令固定长度（32 位 ARM 状态 / 16 位 Thumb 状态）
  2. 加载 - 存储架构：运算指令只能操作寄存器，不能直接读写内存；内存读写必须用专门的 LDR/STR 指令
  3. 指令支持条件执行，几乎所有指令都可以加条件后缀

二、寄存器体系（32 位 ARM）

共 16 个通用寄存器，其中 3 个有特殊功能：

寄存器 别名 核心作用
R0 ~ R3 A1~A4 前 4 个函数参数；返回值放 R0
R4 ~ R11 V1~V8 局部变量寄存器，函数要用必须先保存
R12 IP 临时寄存器
R13 SP 栈顶指针，和 x86 的 ESP 功能一样
R14 LR 链接寄存器，保存函数返回地址（和 x86 最大的区别）
R15 PC 程序计数器，等于当前指令地址 + 8（流水线特性）

核心差异：x86 的返回地址是压在栈里的，ARM 的返回地址默认存在 LR 寄存器里，只有函数嵌套调用时才需要把 LR 压栈保存。

四、AAPCS 调用约定（ARM 标准）

  1. 参数传递：前 4 个参数放 R0~R3，第 5 个及以后压栈
  2. 返回值：32 位返回值放 R0，64 位放 R0+R1
  3. 寄存器保护：R4~R11、SP 是被调用者保护，用了必须恢复；R0~R3、R12、LR 调用者保护
  4. 函数返回：通过 BX LR 返回，嵌套函数需要先把 LR 压栈

五、ARM64（AArch64）简介

  • 寄存器扩展为 X0~X30（64 位），W0~W30 是低 32 位
  • X0~X7 传前 8 个参数，返回值放 X0
  • X30 是 LR（链接寄存器），SP 是栈指针
  • 指令集全面升级，兼容 32 位但差异较大

C 逻辑的汇编识别

拿到一段反汇编，按下面的方法可以快速对应回高级语言逻辑。

1. if-else 分支

特征：cmp/test + 条件跳转，跳转走 else 分支，不跳转走 if 分支

2. for 循环

特征：初始化 → 条件判断 → 循环体 → 增量 → 跳回判断

3. 函数调用

特征：先传参 → call 函数 → 取返回值

  • x86：看到连续 push 就是在传参
  • x64：看到给 RCX/RDX/R8/R9 赋值就是在传参
  • 返回值永远看 EAX/RAX

4. 读汇编通用步骤

  1. 先看函数开头，确定是 32 位还是 64 位、调用约定
  2. 找 cmp/test 和跳转，先把分支结构画出来
  3. 根据调用约定，识别参数和返回值
  4. 结合字符串、API 调用，反推函数功能
  5. 重命名关键变量、函数，逐步还原逻辑
