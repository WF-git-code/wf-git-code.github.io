一、Docker 是什么？为什么需要它？

想象一下，我们把开发好的应用程序及其所有依赖（比如代码、运行时、系统工具、库）全部打包到一个“集装箱”里。这个集装箱是标准化的，可以在任何支持 Docker 的机器上运行，无论这台机器是 Windows、macOS 还是 Linux，也无论它是什么版本。

这个“集装箱”，就是 Docker 容器 (Container)。

Docker 的核心价值：

环境隔离与一致性： 确保开发、测试、生产环境完全一致，告别环境问题。

快速部署与启动： 容器是秒级启动，远快于传统的虚拟机。

资源利用率高： Docker 容器直接运行在宿主机内核上，几乎没有性能损耗。一台物理机可以运行成百上千个容器。

简化运维： 便于迁移、扩展和管理。

DockerVS虚拟机

对比项 虚拟机 (VMware/VirtualBox) Docker 容器
内核 独立完整操作系统内核，占用极高资源 共享宿主机内核，仅打包应用 + 依赖
启动速度 分钟级 秒级启动
资源占用 每台虚拟机分配独立内存 CPU 轻量，可同时跑上百个容器
隔离程度 硬件级强隔离 进程级隔离，性能损耗极低
打包体积 GB 级别（完整系统） MB 级别（仅应用依赖）

Docker C/S架构图

Client：docker 命令行，输入指令发送给守护进程

Daemon (dockerd)：后台服务，管理镜像、容器、网络、存储

Registry：远程镜像仓库（Docker Hub、阿里云镜像仓库）

二、核心概念：镜像、容器、仓库

学习 Docker，必须理解这三个核心概念：

镜像 (Image)：

是什么： 镜像是一个只读的模板，它包含了运行应用程序所需的一切：代码、运行时、库、环境变量和配置文件。

类比： 你可以把镜像想象成一个面向对象编程中的“类”。

**分层特性：**每层只读，可复用，构建时缓存加速

**关键：**镜像不能修改，运行时自动生成一层可读写层

容器 (Container)：

是什么： 容器是镜像的运行实例。镜像是静态的，而容器是动态的。我们对容器的任何修改（如创建、启动、停止、删除）都不会影响到镜像本身。容器删除后，容器内读写层数据默认全部丢失（需数据卷持久化）

类比： 如果镜像是“类”，那么容器就是这个“类”的一个“实例对象”。

仓库 (Repository)：

是什么： 仓库是集中存放镜像文件的地方。

类比： 你可以把它理解成代码界的 GitHub，只不过它存放的是镜像而不是代码。最著名的公共仓库是 Docker Hub。

公共仓库：Docker Hub（官方）、阿里云镜像加速

私有仓库：企业自建 Harbor，存放内部业务镜像

三者关系： 我们从 仓库 pull (拉取) 镜像到本地，然后以这个镜像为模板，run (运行) 起一个或多个容器。

三、什么时候用Docker？

  1. 多人协作开发

    每个人电脑系统、软件版本不一样，有人 Windows、有人 Ubuntu，本地项目环境五花八门，代码拉下来各种依赖报错。统一用 Docker 镜像，所有人环境完全一致。

  2. 本地搭建中间件

    需要临时跑 MySQL、Redis、Nginx、MQ，不想本机安装一堆软件、卸载不干净污染系统，容器用完直接删除，不留残留。

  3. 项目部署上线

    本地能跑，放到云服务器、测试机大概率环境出错；Docker 打包整套环境，服务器只装 Docker，一条命令启动项目。

  4. 多版本软件共存

    比如同时需要 Python3.8、Python3.11，本地安装会冲突，容器分开隔离，互不影响。

  5. CI/CD 自动化流水线

    Git 提交代码后，自动构建 Docker 镜像、自动部署到测试 / 生产服务器。

  6. 临时测试、演示项目

    给别人演示程序，不用对方安装环境，只需要给镜像，一键运行。

不适合：需要独占硬件显卡、深度修改系统内核、桌面 GUI 大型图形程序。
