计算机网络框架

发表于2025-07-27|更新于2026-06-15|计算机网络

一、前置核心：为什么网络要分层？

1. 分层设计的本质思想

计算机网络采用协议栈分层架构，和软件设计里「模块化、解耦、单一职责」思想完全一致。

2. 分层的五大核心目的（标准答案 + 深度原理）

  1. 解耦、降低复杂度

    网络通信逻辑极复杂（寻址、差错、路由、传输、应用交互等），拆分成多层后，

    每层只做一件事

    ，不用一次性处理所有逻辑。

  2. 独立演进、互不影响

    下层协议改动，上层无需修改；上层换应用，下层传输逻辑不变。

    例：手机用 Wi‑Fi、电脑用以太网（二层 / 一层不同），但上层 TCP/IP、HTTP 完全不用改。

  3. 标准化、互通性

    全球设备遵循同一套分层标准，不同厂商、不同系统的设备可以正常通信。

  4. 便于排错、定位问题

    网络故障可以逐层排查：物理层不通 → 二层帧异常 → 三层路由问题 → 四层端口问题 → 应用层协议错误。

  5. 复用性

    底层通信能力被所有上层应用共享，不用每个应用单独实现寻址、差错校验、重传等逻辑。

3. 两大主流分层模型（面试必区分）

业界两套标准，面试必考对比：OSI 七层模型、TCP/IP 五层（四层）模型

后端 / 开发 / 运维 主流使用 TCP/IP 五层模型（实际互联网在用）；OSI 是理论标准。

层级 OSI 七层 TCP/IP 五层（工业主流） TCP/IP 四层（简化叫法）
7 应用层 应用层 应用层
6 表示层 ——（合并到应用层） ——
5 会话层 ——（合并到应用层） ——
4 传输层 传输层 传输层
3 网络层 网络层 网际层（互联网层）
2 数据链路层 数据链路层 网络接口层

统一约定：下文全部以 TCP/IP 五层模型 讲解（后端面试、实际工作唯一标准）。

二、整体数据流：封装 & 解封装

这是分层模型最核心的运行机制，所有层的工作都围绕「封装 / 解封装」展开，面试高频追问。

1. 核心概念

  • 封装（Encapsulation）

    ：发送数据时，

    从上到下逐层加头部首部。

    上层数据 = 下层的数据载荷（Payload）。

  • 解封装（Decapsulation）：接收数据时，从下到上逐层拆掉首部，交给上层。

2. 完整流转流程（客户端 → 服务器）

3. 每层数据单元名称（面试填空题 / 口述考点）

层级 数据单元专业名称
应用层 报文 / 应用数据
传输层 段（Segment）
网络层 包 / 分组（Packet）
数据链路层 帧（Frame）
物理层 比特（Bit）流

面试官常问：TCP 是什么数据单元？IP 是什么？帧属于哪一层？

三、逐层深度讲解

自上而下顺序讲解，每层包含：核心作用、工作原理、核心协议、关键字段、典型设备、高频面试题。

第一层：物理层（Physical Layer）

1. 核心定位

整个网络的物理媒介层，只负责传输二进制比特流（0/1），不理解数据含义。

2. 核心作用

  1. 定义物理接口、线缆、电压、电平、传输速率、编码方式；
  2. 在物理介质上透明传输 0 和 1 比特流；
  3. 负责信号的发送与接收。

3. 传输介质 & 典型设备

  • 传输介质：双绞线、光纤、无线信号（WiFi / 蓝牙）、同轴电缆
  • 典型设备：集线器 HUB、网线、光纤、网卡（物理接口部分）

4. 关键特性 & 原理

  • 无寻址、无差错校验、无协议逻辑，纯硬件信号传递；
  • 信号会衰减、受干扰，远距离传输需要中继器放大信号。

5. 面试高频考点

  1. 物理层作用？传输的数据单元是什么？

    答：负责传输比特流，定义硬件接口与电气标准；数据单元是比特。

  2. 集线器 HUB 工作在哪一层？特点？

    工作在物理层；对所有端口广播数据，无隔离冲突域，所有端口共享带宽。

  3. 网线不通、接口灯不亮，排查哪一层？

    优先排查物理层：网线、网口、供电、硬件故障。

第二层：数据链路层（Data Link Layer）

1. 核心定位

相邻两个节点之间的数据传输（同一局域网内），在物理层比特流基础上，组织成「帧」。

解决问题：如何在直连设备间可靠传递数据、区分设备、纠错、划分边界。

2. 四大核心作用（必背）

  1. 成帧：把上层数据包加上帧头、帧尾，标记一帧数据的开始和结束；
  2. MAC 寻址：使用 MAC 地址 标识局域网内每一台主机 / 网卡；
  3. 差错检测：通过 CRC 校验，检查传输中是否出现比特错误（只检错，不纠错）；
  4. 访问控制 & 冲突处理：同一链路多设备同时发数据产生冲突，使用 CSMA/CD 等机制规避。

3. 核心协议

  1. 以太网协议 Ethernet：局域网最主流帧协议（99% 内网使用）
  2. ARP 协议（地址解析协议）：三层 IP 地址 → 二层 MAC 地址 映射（超级重点）
  3. RARP：反向解析（MAC → IP，现已淘汰）

4. 关键地址：MAC 地址

  • 长度：48 位（6 字节），全球唯一，固化在网卡硬件中；
  • 作用：局域网内部寻址，同一个广播域内依靠 MAC 找到目标主机。

5. 典型设备

  • 交换机 Switch（二层核心设备）：基于 MAC 地址转发帧，隔离冲突域；
  • 网卡（逻辑链路部分）、网桥。

区分考点：HUB（一层，广播） vs 交换机（二层，基于 MAC 单播转发）

6. 重点协议：ARP 协议（原理 + 面试高频）

（1）作用

已知目标 IP 地址，获取对方MAC 地址。

因为：三层通信靠 IP，二层转发靠 MAC，必须做地址转换。

（2）工作原理

  1. 主机先发 ARP 广播帧（广播 MAC：FF:FF:FF:FF:FF:FF），在整个局域网内询问：「这个 IP 对应的 MAC 是谁？」
  2. 局域网内所有主机收到广播，对比自身 IP；
  3. 目标主机回复 ARP 单播响应，把自己 MAC 发给请求方；
  4. 主机本地维护 ARP 缓存表（IP ↔ MAC 映射），一段时间内复用，避免反复广播。

（3）面试深挖题

  1. ARP 广播范围？

    仅当前广播域 / 局域网，路由器会隔绝广播，ARP 广播无法跨网段。

  2. ARP 缓存作用？ARP 欺骗原理？

    ARP 缓存：缓存 IP-MAC 映射，提升效率；

    ARP 欺骗：伪造 ARP 响应包，篡改他人 ARP 缓存，劫持流量。

  3. 跨网段通信，需要 ARP 吗？

    需要。目标不在本网段时，主机把数据发给网关路由器，先解析「网关 IP → 网关 MAC」。

7. 本层面试全集考点

  1. 数据链路层数据单元？帧（Frame）。
  2. MAC 地址作用？和 IP 地址区别？
  3. 交换机工作在哪一层？和 HUB 区别？
  4. ARP 协议工作流程、广播特性、ARP 欺骗。
  5. 帧的 CRC 校验：只检错，不重传纠错。

第三层：网络层（Network Layer）【重中之重，IP 协议主场】

1. 核心定位

实现跨网络、跨网段的主机通信。

二层 MAC 只能局域网互通；网络层负责全球范围内不同网络之间的寻址与路由。

解决核心问题：如何把数据包从源主机，经过多个路由器，送到跨网段的目标主机。

2. 四大核心作用

  1. 逻辑寻址：使用 IP 地址 做全球主机逻辑标识；
  2. 路由选择：选择从源到目标的最佳传输路径（路由器核心工作）；
  3. 分组转发：根据路由表，把数据包从一个网络转发到另一个网络；
  4. 分片与重组：不同链路 MTU（最大传输单元）不同，大数据包自动分片，到达目标再重组。

3. 核心协议（按重要排序）

  1. IP 协议（网际协议）：本层最核心，无连接、不可靠协议
  2. ICMP 协议：互联网控制报文协议，网络探测、差错报告（ping 命令底层）
  3. IGMP 协议：组播管理（了解即可）

3.1 核心：IP 协议（原理 + 字段 + 面试全覆盖）

（1）IP 协议核心特性

  • 无连接：通信前不用建立连接，直接发包；
  • 不可靠：不保证包一定送达、不保证按序、不保证不重复；
  • 只负责「尽力交付」，可靠性交给上层 TCP 实现。

（2）IP 地址作用

  • 全局逻辑地址，用来区分不同网段 + 网段内主机；
  • MAC 是硬件地址（本地局域网），IP 是逻辑地址（全网路由）；
  • 一台主机网卡可以有多个 IP，但 MAC 网卡出厂唯一。

（3）IPv4 基础（面试主流考 IPv4，IPv6 了解即可）

  • 长度：32 位，点分十进制表示 A.B.C.D

  • 结构：

    网络号 + 主机号

    • 网络号：标识当前属于哪一个网段；
    • 主机号：标识网段内某一台主机。

（4）IP 首部关键字段

只讲面试常考关键字段（不用背全部，理解作用）：

  1. 版本（Version）：IPv4 / IPv6

  2. 首部长度：IP 头长度，最小 20 字节

  3. TTL（生存时间）

    • 原理：每经过一台路由器，TTL -1；减到 0 时，数据包被丢弃，防止数据包在环路网络永久游荡。
    • 面试必考：ping、traceroute 底层依靠 TTL 实现路由追踪。

  4. 协议（Protocol）

    ：标识上层使用什么传输协议

    • 6 → TCP
    • 17 → UDP
    • 1 → ICMP

  5. 源 IP 地址、目的 IP 地址：路由转发的核心依据

  6. 标识、标志、片偏移：用于IP 分片（大包拆分传输）

（5）IP 分片原理

不同链路 MTU 不同（以太网默认 MTU=1500 字节）。

  • 当 IP 包大小 > 链路 MTU，IP 层自动分片；
  • 分片在传输路径上的路由器完成；
  • 重组只在目标主机完成，中途路由器不重组。

（6）典型设备

路由器 Router（三层核心设备）

  • 工作在网络层；
  • 根据 IP 地址 + 路由表 转发数据包；
  • 隔离广播域（ARP、二层广播无法穿过路由器）；
  • 连接不同网段、不同网络。

3.2 配套协议：ICMP 协议

（1）作用

网络层控制 / 差错 / 探测协议，辅助 IP 工作，传递网络状态、错误信息、连通性探测。

（2）两大典型应用

  1. ping 命令

    原理：发送 ICMP 回显请求，对方回复回显应答，判断主机是否可达、延迟。

  2. traceroute / tracepath

    原理：利用

    TTL 超时

    特性：

    • 先发 TTL=1 的包，第一台路由器 TTL 归 0，返回 ICMP 超时报文；
    • 逐步增大 TTL，逐个拿到路径上所有路由器地址，追踪完整路由。

（3）面试题

  1. ping 基于什么协议？属于哪一层？

    ICMP 协议，工作在网络层。

  2. ping 不通有哪些原因？（综合排错题）

    物理不通、防火墙禁 ICMP、路由不可达、主机宕机、目标禁 ping。

3.3 网络层整体面试考点汇总

  1. IP 协议特性：无连接、不可靠、尽力交付。
  2. IP 地址与 MAC 地址区别（寻址范围、层级、是否可修改、作用）。
  3. TTL 字段作用，traceroute 原理。
  4. IP 分片场景与规则。
  5. 路由器工作层级、作用；路由器隔离广播域。
  6. ARP 与 IP 配合流程（跨网关通信完整流程）。
  7. ICMP 作用、ping 原理。

第四层：传输层（Transport Layer）

先记定位：网络层负责「主机到主机」；传输层负责「进程到进程」。

两台主机通信，最终是主机上的应用进程在收发数据，传输层实现端口寻址 + 传输控制。

这一层后面你会细学 TCP/UDP，本框架只讲层级定位、通用作用、端口、整体逻辑。

1. 核心作用

  1. 端口寻址

    ：通过

    端口号

    区分一台主机上不同的应用进程；

    • IP 找到主机，端口找到主机内具体程序。

  2. 端到端传输控制

    ：提供两种传输模式：

    • 可靠传输（TCP）
    • 不可靠传输（UDP）

  3. 流量控制、拥塞控制、差错重传（TCP 实现）。

2. 核心概念：端口（Port）

  • 端口号：16 位，范围 0~65535；
  • 知名端口（0~1023）：系统 / 标准服务占用（80=HTTP、443=HTTPS、22=SSH）；
  • 动态端口（1024~65535）：客户端临时使用。

3. 核心协议

  • TCP：面向连接、可靠、字节流
  • UDP：无连接、不可靠、数据报

4. 典型设备

四层交换机、防火墙（可基于端口过滤）。

5. 本层面试基础考点

  1. 网络层和传输层区别？主机到主机 vs 进程到进程。

  2. 端口作用？IP + 端口 组成什么？

    IP + 端口 = Socket 套接字，全网唯一标识一个通信进程。

  3. 为什么需要端口？一台机器多个服务如何区分？

第五层：应用层（Application Layer）

1. 核心定位

直接面向用户 / 应用程序，为应用软件提供网络服务，是分层模型的最顶层。

下层所有能力（寻址、路由、传输）全部为应用层服务。

2. 核心作用

定义应用之间交互的业务报文格式、交互规则。

3. 常见核心协议（后端日常接触最多）

  • HTTP / HTTPS：网页、接口请求（接口开发、网关、反向代理）
  • FTP / SFTP：文件传输
  • DNS：域名解析（域名 → IP）
  • SSH：远程登录
  • SMTP/POP3：邮件

4. 面试基础考点

  1. DNS 工作在哪一层？HTTP 在哪一层？
  2. 域名解析整体流程（结合五层模型串联）。
  3. 应用层数据是如何逐层封装下发的？

四、完整串联：一次 HTTP 请求 = 五层模型全流程

场景：浏览器访问 www.baidu.com，从头到尾串所有层原理，面试官最爱追问。

步骤 1：应用层（DNS + HTTP）

  1. 浏览器先发起 DNS 请求（应用层），把域名解析为目标 IP；
  2. 构造 HTTP 请求报文（应用层数据）。

步骤 2：传输层

HTTP 基于 TCP，添加 TCP 首部，带上「源端口、目标端口」，形成 TCP 段。

步骤 3：网络层

添加 IP 首部，带上「源 IP、目标 IP、TTL、协议号 (TCP=6)」，形成 IP 数据包；

查询路由表，确定下一跳（网关 / 路由器）。

步骤 4：数据链路层

查询 ARP 缓存，拿到下一跳 MAC 地址；

添加帧头（源 MAC、目的 MAC）+ CRC 校验，形成以太网帧。

步骤 5：物理层

帧转为 0/1 比特流，通过网线 / 光纤传输。

转发过程（跨网段）

数据包经过多个路由器（三层设备）：

  • 每到一台路由器：二层帧解封装 → 取出 IP 包 → 查路由表 → 重新封装新二层帧转发；
  • IP 包 TTL 不断减 1；
  • 路由器隔绝二层广播，只转发单播 IP 包。

到达目标服务器（逐层解封装）

物理层 → 链路层（拆帧）→ 网络层（拆 IP）→ 传输层（拆 TCP）→ 应用层（HTTP 服务接收请求）。

五、分层模型 高频面试问答

1. 为什么网络要分层？

分层可以解耦、降低复杂度，每层职责单一；各层独立迭代，下层改动不影响上层；便于标准化、设备互通、故障逐层排查、功能复用。

2. TCP/IP 五层模型分别是哪五层？每层作用 + 数据单元？

  1. 物理层：传输比特流，硬件信号；单元：比特。
  2. 数据链路层：局域网寻址 (MAC)、成帧、差错检测；单元：帧。
  3. 网络层：跨网段寻址 (IP)、路由、分片；单元：分组 / 包；核心协议 IP、ICMP、ARP。
  4. 传输层：进程间通信、端口寻址、可靠 / 不可靠传输；单元：段；协议 TCP/UDP。
  5. 应用层：应用交互、业务协议；单元：报文；协议 HTTP/DNS 等。

3. MAC 地址 和 IP 地址区别？

  1. 层级：MAC 二层，IP 三层；
  2. 作用：MAC 用于局域网内部寻址，IP 用于全网跨网段路由；
  3. 性质：MAC 硬件固化、基本不变；IP 逻辑地址、可动态修改；
  4. 长度：MAC 48 位，IPv4 32 位；
  5. 广播域：MAC 广播被路由器隔离，IP 实现跨路由通信。

4. ARP 协议作用、工作流程？

ARP 属于数据链路层辅助协议，功能是 IP 地址解析为 MAC 地址。

主机发送 ARP 广播，询问目标 IP 对应的 MAC；目标主机单播回应；本地缓存 IP-MAC 映射。ARP 广播仅局限在当前局域网，无法穿越路由器。

5. 路由器、交换机、集线器分别工作在哪一层？区别？

  • 集线器 HUB：物理层，纯信号放大，所有端口广播，共享带宽，不隔离冲突域。
  • 交换机 Switch：数据链路层，基于 MAC 地址转发，隔离冲突域。
  • 路由器 Router：网络层，基于 IP + 路由表转发，隔离广播域，连接不同网段。

6. 网络层 IP 协议特点？

无连接、不可靠、尽力交付；不保证送达、不保证有序、不保证无重复；可靠性由上层 TCP 实现。

7. TTL 作用？

IP 首部 TTL 生存时间，每经过一台路由器减一，为 0 则丢弃数据包；防止路由环路，同时是 traceroute 路由追踪的实现原理。
