五.inline函数

当程序执行函数调用时，系统建立栈帧空间，保护现场，传递参数等，这些工作需要系统时间和空间的开销。

inline函数会在遇到函数名时将函数体粘贴展开到函数名处，不产生栈帧

当代码函数体比较小并且重复多次调用，为了提高效率和可读性使用inline函数

inline函数是一种以空间换时间的做法，省去了调用的额外开销，但当函数体过长或者是递归函数，即便加上inline关键字，也不会在调用点以内联展开该函数。

7.函数重载

在 C++ 中，可以为两个或两个以上的函数提供相同的函数名称，只要满足以下条件之一：

  • 参数类型不同

  • 参数类型相同，但参数的个数不同

  • 参数类型，个数都相同，但不同类型之间的顺序不同

    这种机制被称为函数重载。

1.如果两个函数参数表相同，但返回值类型不同，会被标记编译错误，函数的重复声明。———返回类型

2.参数表的比较过程与形参名无关，只比较形参的类型与个数。———形参名

3.如果在两个函数的参数表中，只有缺省形参不同，则第二个被视为第一个的重复声明———默认类型（缺省参数）

4**.typedef重命名的类型也不能作为区分函数重载的依据**———typedef重命名

5.当一个形参类型有 const 或 volatile 修饰时，如果形参是按值传递方式定义，在识别函数声明是否相同时，并不考虑 const 和 volatile 修饰符。

类型 含义（常性的归属） 示例
顶层 const 修饰指针本身（指针的地址不可改） int* const p：p 的地址不能改，但*p（指向的内容）可改
底层 const 修饰指针指向的对象（内容不可改） const int* p：*p不能改，但 p 的地址可改

如果函数参数为指针/引用时，有 const 或 volatile 修饰时，可以作为区分函数重载的标准

顶层const会被忽略

C++ 标准明确：函数参数的「顶层 const」会被编译器忽略（视为同一类型），而**「底层 const」是类型的一部分，可区分重载**。****

2.decltype

在编译时推导出一个表达式（或表达式的返回类型）的类型，并不会运行表达式或函数

auto和decltype推演的区别？

特性 auto decltype
推导依据 变量的初始化表达式 任意表达式（可以是变量、函数调用、运算式等）
执行性 会执行初始化表达式（给变量赋值） 仅分析表达式的类型，不会执行表达式
类型退化 会发生（数组→指针、const→普通、引用→原值） 不会发生（保留所有类型属性：const、引用、数组等）
核心目标 简化普通变量的声明，追求简洁 精确推导类型，满足模板 / 泛型编程的精确需求

const在两个推演中怎么做？

auto对const的处理依赖于推导的类型是否是引用 / 指针，核心逻辑是：非引用 / 指针时丢弃 const，引用 / 指针时保留 const。

decltype对const的处理极其严格，核心逻辑是：无论表达式是什么类型，始终保留 const 限定符，不会丢弃。此外，decltype还会区分普通变量和带括号的变量表达式

16.面向对象

对象：模拟现实世界的实体

类：实例化为实体

结:类是一组相关的属性（变量）和行为（方法）的集合。是由一个抽象概念设计的产物。

面向对象概念

当对象还未初始化（创建）只是申请了一片空间以一个占位符来表示该对象，大小为1

对象是由一个类型实例化后的具体表现形式，就是具体存在的实体。

成员变量是对象的属性（可以是变量，指针，数组等），属性的值确定对象的状态。

成员函数是对象的方法，确定对象的行为。

状态和行为是对象的主要属性。

  1. 对象的状态又称为对象的静态属性，主要指对象内部所包含的各种信息，也就是变量，每个对象个体都有自己专有的内部变量，这些变量的值标明了对象所处的状态。

  2. 对象的方法（行为）一方面把对象的内部变量包裹，封装，保护起来，使得只有对象自己的方法才能操作对象内部变量，另一方面，对象的方法还是对象与外部环境和其他对象交互，通信的接口，其他对象或外部环境可以通过这个接口来调用对象的方法，操纵对象的行为和改变对象的状态。

对象是现实世界的实体或概念在计算机世界中的抽象表示。具体地，对象是属性和操作的一个集合，用来模拟或影响现实世界中的实体。

面向对象的优点

  1. 对象的数据封装特性彻底消除了传统结构方法中数据与操作分离所带来的种种问题，提高了程序的可复用性和可维护性，降低了程序员保持数据与操作相容的负担。

  2. 对象的数据封装特性还可以把对象的私有数据和公共数据分离开，保护了私有数据，减少了可能的模块间干扰，达到降低程序复杂性，提高可控性的目的。

  3. 对象作为独立的整体具有良好的自恰性，即它可以通过自身定义的操作来管理自己，一个对象的操作可以完成两类功能，一是修改自身的状态，二是向外界发布消息。

  4. 在具有自恰性的同时，对象通过一定的接口和相应的消息机制与外界联系。这个特性与对象的封装性结合在一起，较好地实现了信息的隐蔽。

  5. 通过继承可以很方便地实现应用的扩展和已有代码的重复使用。总结：面向对象程序设计是将数据及数据的操作封装在一起，成为一个不可分割的整体，同时将具有相同特性的实体抽象成为一种新的数据类型 —— 类，通过对象间的消息传递是整个系统运转

对象的关系

  1. 包含：当对象 A 是对象 B 的属性时，称对象 B 包含对象 A.
  2. 继承：当对象 A 是对象 B 的特例时，称对象 A 继承对象 B.
  3. 关联：当对象 A 的引用是对象 B 的属性时，称对象 A 和对象 B 之间是关联关系。所谓对象的引用是指对象的名称，地址，句柄等可以获得和操纵该对象的途径.

19.构造函数与析构函数

定义

数据成员多为私有的，要对它们进行初始化，必须用一个公有函数来进行。同时这个函数应该在且仅在定义对象时自动执行一次。称为构造函数（constructor）。

构造函数用途：1）创建对象，2）初始化对象中的属性，3）类型转换。

特征

构造函数是特殊的公有成员函数（在特殊用途中构造函数的访问限定可以定义成私有或保护），其特征如下:

  1. 函数名与类名相同。

  2. 构造函数无函数返回类型说明。注意是没有而不是 void，即什么也不写，也不可写 void。实际上构造函数有返回值，返回的就是构造函数所创建的对象。

  3. 在程序运行时，当新的对象被建立，该对象所属的类构造函数自动被调用，在该对象生存期中也只调用这一次。不能自己调自己（也可以用定位new再次构建对象属性）

  4. **构造函数可以重载。**严格地讲，类中可以定义多个构造函数，它们由不同的参数表区分，系统在自动调用时按一般函数重载的规则选一个执行。

  5. 构造函数可以在类中定义，也可以在类中声明，在类外定义

  6. 如果类说明中没有给出构造函数，则 C++ 编译器自动给出一个缺省的构造函数: 类名 (void) { }

  7. 并不全是无参（某些特定情况下**(当所有属性公有)会产生有参缺省构造函数**，且对内置类型缺省值为0）

不能在构造函数中使用内存设置（memset）和按位拷贝（memcpy）；虽然无语法错误但可能会产生严重影响会破坏函数的虚表

    限制

    1. 不可臆造新的运算符。

    2. 不能改变运算符原有的优先级、结合性、语法结构，也不能改变运算符操作数的个数。重载后的运算符是一个函数

class IntNumber {
private:
    int value; // 核心数据成员
public:
    // 构造函数
    IntNumber(int v = 0) : value(v) {}

    // 1. 前置++：成员函数实现，无参数，返回引用（支持连续++）
    IntNumber& operator++() {
        value++;
        return *this; // 返回自身引用，支持 ++(++a)
    }//前置++由于不用创建类效率高

    // 2. 后置++：成员函数实现，带int参数（仅作区分），返回值（不能返回引用）
    IntNumber operator++(int) {
        IntNumber temp = *this; // 保存当前值
        value++;
        return temp; // 返回修改前的值
    }

    // 3. 前置--：成员函数实现
    IntNumber& operator--() {
        value--;
        return *this;
    }

    // 4. 后置--：成员函数实现
    IntNumber operator--(int) {
        IntNumber temp = *this;
        value--;
        return temp;
    }

    // 5. 单目-：成员函数实现（取反）
    IntNumber operator-() const {
        return IntNumber(-value);
    }

    // 6. 双目+：友元函数实现（支持左操作数不是类对象的情况，如 10 + a）
    friend IntNumber operator+(const IntNumber& a, const IntNumber& b) {
        return IntNumber(a.value + b.value);
    }

    // 7. < 比较运算符：友元函数实现
    friend bool operator<(const IntNumber& a, const IntNumber& b) {
        return a.value < b.value;
    }

    // 8. > 比较运算符：友元函数实现
    friend bool operator>(const IntNumber& a, const IntNumber& b) {
        return a.value > b.value;
    }

    // 9. += 复合赋值：成员函数实现（修改自身，返回引用）
    IntNumber& operator+=(const IntNumber& other) {
        value += other.value;
        return *this;
    }

    // 10. -= 复合赋值：成员函数实现
    IntNumber& operator-=(const IntNumber& other) {
        value -= other.value;
        return *this;
    }

    // 辅助函数：打印值
    void print() const {
        cout << value << endl;
    }

    // 用于测试的取值函数
    int getValue() const {
        return value;
    }
};

27.类中的静态成员

c语言中的静态变量

类静态成员与静态属性

  1. 类静态成员的定义由关键字static修饰类中的成员，称为类静态成员；该成员为类的所有对象共享，无论实例化多少对象，静态成员仅存在一份公用内存空间，静态数据成员可被当作该类型的全局对象。
  2. 静态属性（静态数据成员的特性）
  • 类中用static修饰的数据成员，称为静态数据成员；
  • 该类型实例化的所有对象，共享系统为静态成员分配的同一个存储空间；
  • 该存储空间的分配时机是程序执行main函数之前，实例化对象时，不会再为静态成员分配空间（静态成员数据不存储在对象自身的空间中）。
  • 静态常性整型（int ,short, char ,long long）可以在类中初始化

设计静态属性的目的是信息共享

其实本质是被限制的全局静态变量。只服务于该类型，且并不算做对象的大小

不受常方法的保护

静态属性不存在this指针，常方法会给this指针加const因为不存在this指针，因此即使在常方法中也可以对其操作

如果将静态属性设计在public中则所有对象可以在类外部访该静态属性，也可以类型名+：：访问符访问，一般设计为私有

静态数据成员的优势

  1. 静态数据成员没有进入程序的全局名字空间，因此不存在与程序中其他全局名字冲突的可能性
  2. 可以实现信息隐藏，静态成员可以是 private 成员，而全局变量不能。
  3. 可以实现信息交流

总结

  1. 设计静态数据成员目的是信息共享，和信息交流.

  2. 类的静态数据成员为所有类对象所共享，不属于某个具体的实例.

  3. 类的静态数据成员必须在类外定义，定义时不添加 static 关键字，不能在构造函数的初始化列表中创建。

  4. 类的静态数据成员类型是 int,short, char, long long, 并且是 const, 可以在类中直接初始化，也可以在类外初始化.

  5. 在类的成员函数中使用静态数据成员，静态数据成员之前没有 this.
