四.引用（别名）

定义a =4Byte=10; b=4Byte=a=10; int&c 对a的空间起了另一个名字叫c，c所指向的空间即为a对应的空间

特点

1.不能定义空引用，定义引用必须给初始化

引用在编译层面会变成自身为常性的指针，如果定义时不初始化，则没机会初始化

2.没有引用的引用

3.引用 不能更改绑定的对象

引用不能为空，当为函数参数类型时，因为此特性不用判空

引用作为形参传递

  • 引用传递：引用是变量的别名，函数的引用参数直接指向实参的内存地址，相当于直接操作原变量，不需要值拷贝。
  • 即使是const int&（常量引用），同样不会发生值拷贝，只是限制了不能通过引用修改原变量
  • 引用传递的优势是效率更高（尤其对于大对象，如自定义的类、数组），还能直接修改实参。
  • 常引用作为函数参数类型时，使用该函数时不能传常量（直接传值或者字符串），要么使用常引用作为参数类型，要不使用右值引用作为参数类型
  • 右值引用参数不能直接接收左值，必须通过std::move()将左值转换为右值引用后才能传递。std::move()是左值转右值引用的工具，使用后需注意原左值的资源安全性（复杂对象慎用）
  • 模板函数：如果是满足条件的T&&（万能引用），可以接收左值和右值（底层依赖类型推导和引用折叠规则）。

底层逻辑：当传递左值a时，模板的类型推导会将T推导为int&，此时T&&变为int& &&，根据 C++ 的引用折叠规则，最终会变成int&（左值引用）；当传递右值10时，T推导为int，T&&就是int&&（右值引用）。

引用与const

常引用只可以读，普通引用可以改值

常引用为万能引用

正常情况下，临时变量的生命周期只在当前语句结束后就会销毁，但 C++ 标准规定：当const引用绑定到临时变量时，临时变量的生命周期会被延长，和引用的生命周期保持一致（直到引用出作用域被销毁）。这也是const引用绑定字面量后能安全使用的核心原因。

右值引用

左值：能够取地址

右值：不能取地址

普通引用只能引用左值

常引用与右值引用的区别

同样都会在底层创建一个临时变量来存储，然后给这个临时变量起别名。

常引用不可改变值，右值引用可以改变值

右值引用不能引用左值，在c11标准中对于一个变量除了有类型这个身份外，还有一个左右值的设定

指针的引用

不存在指向引用的指针：因为引用是变量的别名，没有独立的内存地址，指针无法存储 “引用的地址”。

引用和指针的区别

语法层面引用时指针的语法糖

不同点

1.从语法规则上讲：指针变量存储某个实例（变量或对象）的地址，引用时某个实例的别名。

2.程序为指针变量分配内存区域，而不为引用分配内存区域

3.指针要解引用，别名直接使用

4.指针变量的值可以发生改变，存储不同实例的地址，引用在定义时就被初始化，之后无法改变（不能是其他实例的引用）

5.指针可以为空，没有空引用

6.指针变量作为形参时需要测试合法性（判空），引用不需要

7.对指针”sizeof”得到的是指针的大小，对引用使用“sizeof”得到的是变量的大小

8.指针可以无限分级，引用没有分级的概念

9.指针++（指针自身存储的地址自增）和引用++（被引用变量的值自增）效果不同，但指针++和(指针的引用)++效果相同（前提时同类型）

相同点：

绝不容许将局部变量以指针/引用的形式返回，若要使用则需要变量生存期大于函数

存在值残留，栈帧空间没消失又被拿过来

汇编层面

底层上讲引用就是指针

引用在编译层面会变成自身为常性的指针

可模拟为

注意：并不代表着引用在任何场景都可以用 * const替换

引用的语法糖需要底层的指针结构来 “落地”，而T * const是最适合的指针类型，但并不代表任何场景都能用T * const来模拟，编译器会根据引用的操作场景，灵活选择 “是否模拟” 或 “其他处理”

从计算机硬件的角度，CPU 只能识别内存地址和存储在地址中的数据，根本不存在 “别名” 这种概念。引用的 “别名” 是 C++ 为了简化编程、提升代码可读性设计的高层语法抽象（语法糖），编译器必须把这个抽象转换成硬件能理解的底层操作。

而指针是 C++ 中直接对应 “内存地址” 的底层结构，编译器可以很方便地用指针来实现引用的逻辑：

  • 当你写int& a = b;时，编译器在底层会悄悄创建一个int* const指针（比如int* const __ref_a = &b;，这个指针是编译器内部的，你无法访问）。
  • 当你写a = 100;时，编译器会转换成*__ref_a = 100;（解引用指针修改目标值）。
  • 当你写&a时，编译器会转换成__ref_a（直接返回指针存储的地址，也就是b的地址，所以你看不到引用的独立地址）。
