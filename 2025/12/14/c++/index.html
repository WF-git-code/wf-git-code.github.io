引用作为形参传递

  • 引用传递：引用是变量的别名，函数的引用参数直接指向实参的内存地址，相当于直接操作原变量，不需要值拷贝。
  • 即使是const int&（常量引用），同样不会发生值拷贝，只是限制了不能通过引用修改原变量
  • 引用传递的优势是效率更高（尤其对于大对象，如自定义的类、数组），还能直接修改实参。
  • 常引用作为函数参数类型时，使用该函数时不能传常量（直接传值或者字符串），要么使用常引用作为参数类型，要不使用右值引用作为参数类型
  • 右值引用参数不能直接接收左值，必须通过std::move()将左值转换为右值引用后才能传递。std::move()是左值转右值引用的工具，使用后需注意原左值的资源安全性（复杂对象慎用）
  • 模板函数：如果是满足条件的T&&（万能引用），可以接收左值和右值（底层依赖类型推导和引用折叠规则）。

底层逻辑：当传递左值a时，模板的类型推导会将T推导为int&，此时T&&变为int& &&，根据 C++ 的引用折叠规则，最终会变成int&（左值引用）；当传递右值10时，T推导为int，T&&就是int&&（右值引用）。

引用和指针的区别

语法层面引用时指针的语法糖

不同点

1.从语法规则上讲：指针变量存储某个实例（变量或对象）的地址，引用时某个实例的别名。

2.程序为指针变量分配内存区域，而不为引用分配内存区域

3.指针要解引用，别名直接使用

4.指针变量的值可以发生改变，存储不同实例的地址，引用在定义时就被初始化，之后无法改变（不能是其他实例的引用）

5.指针可以为空，没有空引用

6.指针变量作为形参时需要测试合法性（判空），引用不需要

7.对指针”sizeof”得到的是指针的大小，对引用使用“sizeof”得到的是变量的大小

8.指针可以无限分级，引用没有分级的概念

9.指针++（指针自身存储的地址自增）和引用++（被引用变量的值自增）效果不同，但指针++和(指针的引用)++效果相同（前提时同类型）

相同点：

绝不容许将局部变量以指针/引用的形式返回，若要使用则需要变量生存期大于函数

存在值残留，栈帧空间没消失又被拿过来

汇编层面

底层上讲引用就是指针

引用在编译层面会变成自身为常性的指针

可模拟为

注意：并不代表着引用在任何场景都可以用 * const替换

引用的语法糖需要底层的指针结构来 “落地”，而T * const是最适合的指针类型，但并不代表任何场景都能用T * const来模拟，编译器会根据引用的操作场景，灵活选择 “是否模拟” 或 “其他处理”

从计算机硬件的角度，CPU 只能识别内存地址和存储在地址中的数据，根本不存在 “别名” 这种概念。引用的 “别名” 是 C++ 为了简化编程、提升代码可读性设计的高层语法抽象（语法糖），编译器必须把这个抽象转换成硬件能理解的底层操作。

而指针是 C++ 中直接对应 “内存地址” 的底层结构，编译器可以很方便地用指针来实现引用的逻辑：

  • 当你写int& a = b;时，编译器在底层会悄悄创建一个int* const指针（比如int* const __ref_a = &b;，这个指针是编译器内部的，你无法访问）。
  • 当你写a = 100;时，编译器会转换成*__ref_a = 100;（解引用指针修改目标值）。
  • 当你写&a时，编译器会转换成__ref_a（直接返回指针存储的地址，也就是b的地址，所以你看不到引用的独立地址）。

五.inline函数

当程序执行函数调用时，系统建立栈帧空间，保护现场，传递参数等，这些工作需要系统时间和空间的开销。

inline函数会在遇到函数名时将函数体粘贴展开到函数名处，不产生栈帧

当代码函数体比较小并且重复多次调用，为了提高效率和可读性使用inline函数

inline函数是一种以空间换时间的做法，省去了调用的额外开销，但当函数体过长或者是递归函数，即便加上inline关键字，也不会在调用点以内联展开该函数。

注意：inline函数不建议定义和声明分离，会报编译错误，因为在声明后，再次到.cpp中找函数执行，函数名会被展开，找不到函数地址，因此可以在.h中直接定义并实现该函数

不要超过5~6行代码

使用inline处理效率高。如果函数的执行开销大于开栈清栈开销，则使用普通方式处理

六.缺省参数

c++支持；c语言不支持

  1. 背景：一般函数调用时实参个数与形参相同，C++ 允许定义缺省参数函数，调用时实参个数可与形参不同。
  2. 缺省参数的定义：在定义函数时为形参指定缺省值（默认值）。
  3. 调用规则：调用该类函数时，对于缺省参数，可给出实参值（此时传递实参），也可不给出（此时使用缺省值）。
  4. 缺省实参的要求：
    • 不一定是常量表达式，可为任意表达式，甚至能通过函数调用给出；
    • 若缺省实参是任意表达式，函数每次被调用时该表达式会重新求值，但表达式必须有意义。

注意：在定义是缺省参数一定是从右向左依次赋值；在使用时缺省从左向右依次赋值

在多文件中，只在.h函数的声明中定义缺省参数，在 “.cpp ”实现使用普通方式定义函数

7.函数重载

在 C++ 中，可以为两个或两个以上的函数提供相同的函数名称，只要满足以下条件之一：

  • 参数类型不同

  • 参数类型相同，但参数的个数不同

  • 参数类型，个数都相同，但不同类型之间的顺序不同

    这种机制被称为函数重载。

1.如果两个函数参数表相同，但返回值类型不同，会被标记编译错误，函数的重复声明。———返回类型

2.参数表的比较过程与形参名无关，只比较形参的类型与个数。———形参名

3.如果在两个函数的参数表中，只有缺省形参不同，则第二个被视为第一个的重复声明———默认类型（缺省参数）

4**.typedef重命名的类型也不能作为区分函数重载的依据**———typedef重命名

5.当一个形参类型有 const 或 volatile 修饰时，如果形参是按值传递方式定义，在识别函数声明是否相同时，并不考虑 const 和 volatile 修饰符。

如果函数参数为指针/引用时，有 const 或 volatile 修饰时，可以作为区分函数重载的标准

使用函数重载时，尽量不要使用缺省参数

定义引用时不要定义值类型

new的函数方式的调用

new 的 “普通调用” 与 “直接调用 operator new 函数” 的区别
核心差异在于是否包含 “对象构造 / 析构” 逻辑：
①普通 new 调用（如new int）是 C++ 的new运算符，会完成两步操作：
先调用::operator new分配内存；
再调用对象的构造函数（若为类类型）。
对应的delete运算符也会先调用析构函数，再调用::operator delete释放内存。
②直接调用::operator new函数仅完成 “内存分配”，等价于 C 语言的malloc，不会调用构造函数；对应的::operator delete也仅释放内存，等价于 C 的free，不会调用析构函数。

定位new

对于内置类型 new/delete/malloc/free 可以混用。区别:

1、new/delete 是 C++ 中的运算符。malloc /free 是函数。

2、malloc 申请内存空间时，手动计算所需大小，new 只需类型名，自动计算大小；

3、malloc 申请的内存空间不会初始化，new 可以初始化；

4、malloc 的返回值为 void*，接收时必须强转，new 不需要；

5、malloc 申请内存空间失败时，返回的是 NULL，使用时必须判空；new 申请内存空间失败时抛出异常，所以要有捕获异常处理程序；

2.decltype

在编译时推导出一个表达式（或表达式的返回类型）的类型，并不会运行表达式或函数

auto和decltype推演的区别？

特性 auto decltype
推导依据 变量的初始化表达式 任意表达式（可以是变量、函数调用、运算式等）
执行性 会执行初始化表达式（给变量赋值） 仅分析表达式的类型，不会执行表达式
类型退化 会发生（数组→指针、const→普通、引用→原值） 不会发生（保留所有类型属性：const、引用、数组等）
核心目标 简化普通变量的声明，追求简洁 精确推导类型，满足模板 / 泛型编程的精确需求

const在两个推演中怎么做？

auto对const的处理依赖于推导的类型是否是引用 / 指针，核心逻辑是：非引用 / 指针时丢弃 const，引用 / 指针时保留 const。

decltype对const的处理极其严格，核心逻辑是：无论表达式是什么类型，始终保留 const 限定符，不会丢弃。此外，decltype还会区分普通变量和带括号的变量表达式
